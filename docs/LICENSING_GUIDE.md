## AGPLv3 License (FREE)

### You Can Use AGPLv3 If You:

✅ Use for personal projects
✅ Use for research and education
✅ Build open source applications
✅ Are willing to share your modifications
✅ Provide source code to network service users

### AGPLv3 Key Requirements:

1. **Source Code Disclosure**: Must provide source code to users
2. **Same License**: Derivative works must be AGPLv3
3. **Network Use Clause**: Must provide source even for SaaS applications
4. **Attribution**: Must preserve copyright notices
5. **No Additional Restrictions**: Cannot add proprietary terms

### Example AGPLv3 Use Cases:

- Academic research projects
- Open source SaaS platforms
- Educational tools and demos
- Personal hobby projects
- Open source commercial products (rare)

## Commercial License (PAID)

### You Need Commercial License If You:

⚠️ Use in closed-source products
⚠️ Provide as SaaS without releasing code
⚠️ Cannot comply with AGPL terms
⚠️ Want to modify without sharing changes
⚠️ Need to sublicense or redistribute

### Commercial License Benefits:

✅ **No Source Code Disclosure**: Keep your code proprietary
✅ **Modify Freely**: Change code without sharing
✅ **Redistribute**: Include in your products
✅ **Support Options**: Get technical assistance
✅ **Legal Protection**: Indemnification available

### Example Commercial Use Cases:

- Proprietary financial trading systems
- Closed-source scientific software
- Commercial simulation products
- SaaS platforms (without open sourcing)
- Embedded in proprietary hardware

## Frequently Asked Questions

### General Questions

**Q: Why dual licensing?**
A: To support both open source community and sustainable development. Open source users get free access, commercial users fund continued development.

**Q: Can I start with AGPL and switch to commercial later?**
A: Yes! Your past AGPL usage remains valid. Purchase commercial license for future use.

**Q: What if I'm not sure which license I need?**
A: Contact us! We'll help you determine the right license for your use case.

### AGPLv3 Questions

**Q: Can I use AGPLv3 for my startup?**
A: Yes, if you're building an open source product and will release your source code.

**Q: What if I modify the code?**
A: Under AGPL, you must share your modifications under AGPL too.

**Q: Can I use this in a SaaS product under AGPL?**
A: Yes, but you must provide your complete source code to users.

**Q: What's the difference between AGPL and GPL?**
A: AGPL requires source code disclosure even for network services (SaaS). GPL only requires disclosure when distributing software.

## Compliance

### For AGPLv3 Users:

1. Include AGPL license text in your distribution
2. Provide source code access to users
3. Document your modifications
4. Maintain copyright notices
5. Include "How to get source code" notice in your UI/documentation

### For Commercial License Users:

1. Keep license certificate on file
2. Don't exceed licensed developer count or scope
3. Renew before expiration (for annual licenses)
4. Report usage if audit clause applies
5. Maintain confidentiality of license terms
